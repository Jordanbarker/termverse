# Term Crunch (`@tt/term-crunch`)

A terminal-skills challenge game built **entirely on `@tt/core`** to prove the engine-reuse pattern. basePath `/termverse/term-crunch`. Run via `npm run dev:crunch` / `npm run build:crunch`.

> Monorepo-wide context (`@tt/core`, tech stack, deploy) lives in the repo-root `.claude/CLAUDE.md`. The shared engine skills `tmux` and `commands` (both apply to `@tt/core`) are at root; this app's own skill is `apps/term-crunch:challenges`.

**It does NOT import termoil's story code** — none of termoil's `gameStore`/`COMPUTERS`, no story flags, no emails/piper/chip. (term-crunch has its own lean `useGameStore` in `state/gameStore.ts`; the rule is about not reaching into the sibling app.) Keep it that way: anything shared must come through `@tt/core`, and `@tt/core/terminal/paneTypes` helpers must stay pure and store-agnostic so both apps share them. It *does* parse a player-editable `~/.zshrc` + `~/.tmux.conf` (Settings modal), but only via the shared `@tt/core` parsers (`terminal/tmuxConfig`, `terminal/envParse`) — never termoil's dotfiles or env-seeding code.

## Structure (`apps/term-crunch/src/`)

- **`challenges/`** — declarative challenge framework. `types.ts` (`Challenge`, `Step`, `ChallengeSnapshot`), `registry.ts` (ordered `CHALLENGES[]`), `categories.ts` (selectable tracks derived from each challenge's `type` via `getCategory`/`SELECTABLE_CATEGORIES`), `scheduler.ts` (pure SM-2-lite spaced-repetition scheduler behind the completion-gate self-grades and the `review` command), and one file per challenge (`panes-split.ts`, `panes-grid.ts`, `panes-cleanup.ts`, `panes-resize.ts`, `panes-resize-rows.ts`, `panes-resize-corner.ts`, `windows-create.ts`, `git-first-commit.ts`, `git-unstage.ts`, `git-stash.ts`, `git-pull-ff.ts`, `git-rebase.ts`, `rm-bomb.ts`, `chmod-perms.ts`, `mv-organize.ts`, `env-export.ts`, `alias-shortcut.ts`, `copy-mode-yank.ts`, `sessions-detach-attach.ts`, `sessions-juggle.ts`, and the Vim track `vim-first-edit.ts`, `vim-delete-lines.ts`, `vim-fix-word.ts`, `vim-yank-paste.ts`, `vim-search-fix.ts`, `vim-reorder.ts`). Categories (tracks) are `all/tmux/git/fs/vim`. See the `apps/term-crunch:challenges` skill.
- **`state/gameStore.ts`** — lean Zustand store: ports `@tt/core` window/pane actions, drives win-detection via `checkCompletion()`, tracks per-challenge timing/`bestTimes` + spaced-repetition `reviewStats` (graded at the gate, drives review sessions), and holds the player's `zshrc`/`tmuxConf` (persisted via zustand `persist`, `name: "term-crunch-progress"`). Win-detection and review-gate mechanics live in the `apps/term-crunch:challenges` skill.
- **`components/`** — `GameShell.tsx` (layout), `TabManager.tsx` (thin adapter over the shared `@tt/core/terminal/useTabManager` hook, which owns the xterm runtimes, prefix/repeat/copy-mode pipeline, and `~/.tmux.conf` parsing; the app supplies store actions via `TabManagerAdapter` and the challenge-continue gate, per-pane `LineEditor`s, and editor/pager sessions via `TabManagerExtensions` — see the root `tmux` skill), `TabBar.tsx` (thin wrapper over the shared `@tt/core/components/TmuxStatusBar`), `SettingsModal.tsx` (editors for `~/.zshrc` + `~/.tmux.conf`), `ChallengePanel.tsx` (category/challenge dropdowns, Restart, Settings) + the target readouts `SchematicView.tsx` / `WindowStripView.tsx` / `FsTreeView.tsx`, and `VimCheatSheet.tsx` (beginner key reference shown for `type: "vim"` challenges; lists only keys the core vim engine supports — keep in sync with core `HELP_TEXTS.vim`).
- **`engine/commands/navigation.ts`** — app-local keyboard-first challenge-navigation builtins (`challenges`/`goto`/`next`/`prev`/`track`/`review`), always available regardless of the challenge allowlist and listed by `help` as cyan meta commands (core `registerMetaCommands`). Handlers only queue a pending navigation; `runLine` applies it after committing shell state so `loadChallenge`'s fresh seed isn't clobbered.
- **`hooks/useTerminal.ts`** — wires xterm to the store; `runLine` delegates the chain/pipe loop to the shared `@tt/core/commands/runPipeline` (no redirection or file-read events — those are termoil opt-ins); side-effect-imports `lib/availabilityPolicy` and feeds `getAvailableCommands(CRUNCH_MACHINE)` to the shared `@tt/core/suggestions` engine so ghost-text/TAB honor the allowlist. Only receives complete, submitted lines — syntactically incomplete input (unterminated quote, trailing `\`/`\|`/`&&`/`\|\|`) is held in a zsh-style secondary prompt by the shared `@tt/core/terminal/lineEditor` `LineEditor` before it ever reaches `runLine`.
- **`lib/`** — `machine.ts` (`CRUNCH_MACHINE`/`HOME_DIR`), `availabilityPolicy.ts` (registers the `@tt/core` `AvailabilityPolicy` enforcing each challenge's `commands` allowlist — see the challenges skill), `seed.ts` (`buildBaseFs()` + `applyConfigs`), `defaultConfigs.ts` (`DEFAULT_ZSHRC`/`DEFAULT_TMUX_CONF`), `gitState.ts` (git-state reader for validators), `paneCompare.ts` (structural pane-tree compare, ignores ids), `windowLabel.ts` (wrapper over `@tt/core/terminal/windowLabel` baking in the `crunch` host).

## Window/pane UX

**tmux session lifecycle**: every challenge starts attached to session `"0"` (`loadChallenge` seeds `tmuxAttachedSession`/`tmuxDetachedSessions`; transient, never persisted). The `tmux` command (core builtin, in `ALWAYS_AVAILABLE`) + `<prefix> d` detach/attach/kill via the store's `applyTmuxAction`; while detached the status bar, dividers, chords, and copy mode are gone (bare single shell) and `checkCompletion` is **skipped** so the bare-shell layout can't falsely satisfy a pane/window predicate — unless the challenge sets `checkWhileDetached` (session-lifecycle challenges, whose predicates read `snapshot.tmux` instead of the pane tree). See the root `tmux` skill for the shared model.

Mirrors the live game's multi-window model, driven by a **player-editable `~/.tmux.conf`** (Settings modal, defaulting to `DEFAULT_TMUX_CONF`): the prefix key, status-bar theme, and vim pane focus/resize binds come from the shared `@tt/core/terminal/tmuxConfig` parsers (default prefix Ctrl+Space). Built-in chords `<prefix> c/n/p/1-9/r` (new/cycle/jump/rename; `.`/`,` alias next/prev) and pane chords `| - o x` + arrow focus remain as fallbacks for keys the conf doesn't bind. The status line (`@tt/core/components/TmuxStatusBar`) and rename-window prompt (`@tt/core/terminal/useRenameWindowPrompt`) are shared with the live game; only keep-alive logic still mirrors `TabManager.tsx`. The `~/.zshrc` (also Settings-editable, defaulting to `DEFAULT_ZSHRC`) is seeded into every challenge's fs and its aliases/exports activated for the session.
