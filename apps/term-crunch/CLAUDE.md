# Term Crunch (`@tt/term-crunch`)

A terminal-skills challenge game built **entirely on `@tt/core`** to prove the engine-reuse pattern. basePath `/termverse/term-crunch`. Run via `npm run dev:crunch` / `npm run build:crunch`.

> Monorepo-wide context (`@tt/core`, tech stack, deploy) lives in the repo-root `.claude/CLAUDE.md`. The shared engine skills `tmux` and `commands` (both apply to `@tt/core`) are at root; this app's own skill is `apps/term-crunch:challenges`.

**It does NOT import termoil's story code** — none of termoil's `gameStore`/`COMPUTERS`, no story flags, no emails/piper/chip. (term-crunch has its own lean `useGameStore` in `state/gameStore.ts`; the rule is about not reaching into the sibling app.) Keep it that way: anything shared must come through `@tt/core`, and `@tt/core/terminal/paneTypes` helpers must stay pure and store-agnostic so both apps share them. It *does* parse a player-editable `~/.zshrc` + `~/.tmux.conf` (Settings modal), but only via the shared `@tt/core` parsers (`terminal/tmuxConfig`, `terminal/envParse`) — never termoil's dotfiles or env-seeding code.

## Structure (`apps/term-crunch/src/`)

- **`challenges/`** — declarative challenge framework. `types.ts` (`Challenge`, `Step`, `ChallengeSnapshot`), `registry.ts` (ordered `CHALLENGES[]`), `categories.ts` (selectable tracks derived from each challenge's `type` — `all`/`panes`/`git`/`fs` — via `getCategory`/`SELECTABLE_CATEGORIES`; the store's `challengeIndex` is **category-relative**), and one file per challenge (`panes-split.ts`, `windows-create.ts`, `git-first-commit.ts`, `git-stash.ts`, `git-rebase.ts`, `rm-bomb.ts`, `chmod-perms.ts`). See the `apps/term-crunch:challenges` skill.
- **`state/gameStore.ts`** — lean Zustand store. Ports `@tt/core` window/pane actions (`splitPane`/`focusDirection`/`cyclePane`/`resizePane`/`nudgePaneRatio`/`newWindow`/`selectWindow`/`cycleWindow`/`closeWindow`/`renameWindow`) and drives win-detection via `checkCompletion()` (called after every command + pane mutation). `restartChallenge()` re-seeds the current challenge (recover from destructive dead-ends like `rm -rf`). Holds the player's `zshrc`/`tmuxConf` strings (Settings modal); `loadChallenge` writes them into the fresh fs via `applyConfigs` and parses the zshrc into `envVars`/`aliases` (`parseEnvAssignments`/`parseAliases`), and `setConfigs`/`resetConfigs` re-apply edits live (re-seed fs + re-derive env/aliases) without a challenge reset. Tracks per-challenge timing: `challengeStartTime` is stamped on `loadChallenge`, and when the last step passes `checkCompletion()` records `lastElapsedMs`/`lastWasBest` and updates `bestTimes` (challengeId → best ms). Wrapped in `persist` middleware (`name: "term-crunch-progress"`, `partialize` to `bestTimes` + `activeCategory` + `zshrc` + `tmuxConf`) so personal bests, the selected track, and the dotfiles survive a refresh; the storage getter falls back to an in-memory no-op when `localStorage` is unavailable (node test env). `ChallengePanel` renders the live timer/best via `formatElapsed` from `@tt/core/lib/format`.
- **`components/`** — `GameShell.tsx` (layout), `TabManager.tsx` (trimmed xterm renderer; lifts window/prefix state, keeps non-active windows alive via `display:none`; reads the player's `~/.tmux.conf` from the store and applies it live — `parseTmuxPrefix`/`parseTmuxTheme`/`parseTmuxBindings` drive the prefix char, status-bar theme, and vim pane focus/resize binds incl. tmux `-r` repeat, read through refs so the per-pane `onData` closure sees fresh config; line editing — arrows, mid-line cursor, Home/End, word-skip, Ctrl+A/E/U/K/L/W/D, ghost/TAB completion — is delegated to the shared `@tt/core/terminal/lineEditor` `LineEditor`, one instance per pane, fed app thunks `getContext`/`getHistory`/`getPrompt`; the rename-window prompt is the shared `@tt/core/terminal/useRenameWindowPrompt` hook), `TabBar.tsx` (a thin wrapper over the shared `@tt/core/components/TmuxStatusBar` — supplies store-derived props + the parsed `StatusBarTheme` from `TabManager` as a prop, and the plain `+` button as the `trailing` slot), `SettingsModal.tsx` (⚙ editors for `~/.zshrc` + `~/.tmux.conf` → `setConfigs`/`resetConfigs`), `ChallengePanel.tsx` (category dropdown via `selectCategory` + a per-track challenge dropdown + a Restart-challenge button + the ⚙ Settings button) + `SchematicView.tsx` (pane-tree target readout) + `WindowStripView.tsx` (window-strip target readout for window-count challenges) + `FsTreeView.tsx` (filesystem-subtree readout for `fs` challenges, rendering each node's permission string).
- **`hooks/useTerminal.ts`** — wires xterm to the store; side-effect-imports `lib/availabilityPolicy` so the per-challenge command allowlist is active. `buildSuggestionContext()` feeds the shared `@tt/core/suggestions` engine with `getAvailableCommands(CRUNCH_MACHINE)`, so ghost-text + TAB completion honor the current challenge's allowlist (same source `help` filters through).
- **`lib/`** — `machine.ts` (`CRUNCH_MACHINE`/`HOME_DIR`), `availabilityPolicy.ts` (registers an `@tt/core` `AvailabilityPolicy` driven by the current challenge's `commands` allowlist — hides off-list commands from `help`/suggestions and blocks them at run time with a friendly hint; `help`/`clear`/`man` always allowed — `man` self-scopes via core `man.ts`, so off-list commands still return "No manual entry"), `seed.ts` (`buildBaseFs()` + `applyConfigs(fs, zshrc, tmuxConf)` which writes the player's dotfiles into a fresh fs), `defaultConfigs.ts` (`DEFAULT_ZSHRC`/`DEFAULT_TMUX_CONF` — working defaults that reproduce the built-in look/binds), `gitState.ts` (git readout for the panel), `paneCompare.ts` (structural pane-tree compare, ignores ids), `windowLabel.ts` (a thin wrapper over `@tt/core/terminal/windowLabel` that bakes in the `crunch` host + `USERNAME`; used by `TabBar` and `WindowStripView`).

## Window/pane UX

Mirrors the live game's multi-window model, driven by a **player-editable `~/.tmux.conf`** (Settings modal, defaulting to `DEFAULT_TMUX_CONF`): the prefix key, status-bar theme, and vim pane focus/resize binds come from the shared `@tt/core/terminal/tmuxConfig` parsers (default prefix Ctrl+Space). Built-in chords `<prefix> c/n/p/1-9/r` (new/cycle/jump/rename; `.`/`,` alias next/prev) and pane chords `| - o x` + arrow focus remain as fallbacks for keys the conf doesn't bind. The status line (`@tt/core/components/TmuxStatusBar`) and rename-window prompt (`@tt/core/terminal/useRenameWindowPrompt`) are shared with the live game; only keep-alive logic still mirrors `TabManager.tsx`. The `~/.zshrc` (also Settings-editable, defaulting to `DEFAULT_ZSHRC`) is seeded into every challenge's fs and its aliases/exports activated for the session.
